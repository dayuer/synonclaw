# 📊 验收标准 — SynonClaw 官网

## AC-01：全局导航
- [ ] 导航栏含 5 个链接且点击可路由跳转
- [ ] 当前页面链接高亮
- [ ] 移动端（≤768px）导航折叠为汉堡菜单
- [ ] 滚动后导航栏添加半透明模糊背景

## AC-02：首页 Hero
- [ ] 深色背景全屏展示
- [ ] 主标题 + 副标题可见
- [ ] 双 CTA 按钮可点击（跳转 /desk 和 /enterprise）
- [ ] 元素具有入场渐入动画

## AC-03：首页核心特性区
- [ ] 展示三个特性卡片（数据安全 / 开箱即拉群 / 人机协同）
- [ ] 每个卡片含图标、标题、描述
- [ ] 滚动进入视口时有渐入效果

## AC-04：个人版 (Desk) 页面
- [ ] 产品 Hero 区含渲染图和预定 CTA
- [ ] 硬件配置以表格/网格呈现（非大段文字）
- [ ] 至少展示 2 个使用场景

## AC-05：企业版 (Enterprise) 页面
- [ ] 机柜 Hero 区含渲染图和"预约演示" CTA
- [ ] 展示三大企业级特性
- [ ] 点击 CTA 弹出线索收集表单

## AC-06：线索收集表单
- [ ] 模态框含 5 个字段（企业名称/节点数/联系人/电话/邮箱）
- [ ] 必填字段未填时阻止提交并显示提示
- [ ] 邮箱/电话格式验证
- [ ] 提交后显示成功状态

## AC-07：开发者网络页面
- [ ] 展示三步成长路径（学习/认证/接单）
- [ ] 含"注册开发者账号" CTA

## AC-08：软件生态页面
- [ ] 展示 OpenClaw → 云兔 工作流程图
- [ ] 流程图含数据流向可视化

## AC-09：响应式与SEO
- [ ] 所有页面在 375px 宽度下无横向溢出
- [ ] 每个页面含独立 title 和 meta description
- [ ] 交互元素均有唯一 ID
