# SynonClaw Admin — 业务场景拆解

> 📊 BA 产出 | 复杂度：L

## 1. 领域术语表

| 术语 | 定义 | 属性 |
|------|------|------|
| 产品 (Product) | SynonClaw 硬件终端型号 | name, type(desktop/rack), specs, price, status(active/inactive) |
| 客户 (Customer) | 购买/使用 SynonClaw 的组织或个人 | name, contact, type(ToB/ToC), status(lead/signed/churned), deviceCount |
| 订单 (Order) | 硬件购买交易记录 | orderNo, customerId, productId, amount, status(pending/producing/shipped/completed), createdAt |
| 开发者 (Developer) | 认证技术人才 | name, school, skills[], certLevel(junior/mid/senior), status(active/inactive), taskCount |
| 统计卡片 (StatCard) | Dashboard 上的关键指标 | label, value, icon, trend |

## 2. 业务链路图

### 主链路 (Happy Path)

```
管理员登录(模拟) → Dashboard 概览 → 聚焦某模块 → 列表浏览 → 查看详情 → 执行操作 → 返回列表
```

### 各模块操作链路

```
产品管理: 列表 → 新增/编辑表单 → 保存 → 列表刷新; 列表 → 切换上架状态
客户管理: 列表 → 搜索/筛选 → 查看详情 → 关联设备/订单
订单管理: 列表 → 查看详情 → 推进状态 → 状态变更历史更新
开发者管理: 列表 → 查看详情 → 变更认证等级
```

### 异常链路

```
空数据: 模块无数据 → 展示空状态占位图
无效路由: /admin/unknown → 404 回退到 Dashboard
```

## 4. 边界条件清单

| # | 边界 | 处理策略 |
|---|------|---------|
| B1 | 列表空状态 | 所有列表页统一空状态组件（图标 + 说明文字 + 操作引导） |
| B2 | 长文本溢出 | 表格单元格使用 `text-overflow: ellipsis`，详情页完整展示 |
| B3 | 侧边栏在窄屏 | ≤768px 自动隐藏，顶栏出现汉堡菜单触发浮层 |
| B4 | 表单必填校验 | 产品名、价格为必填，空值提交时表单内联错误提示 |
| B5 | 订单状态流转合法性 | 只能向前推进(pending→producing→shipped→completed)，不可跳跃或回退 |
| B6 | 数字格式化 | Dashboard 大数字使用千分位分隔符 |
| B7 | 路由守卫 | `/admin/xxx` 无效子路径 → 重定向到 `/admin` |

## 7. 技术风险清单 (L 级)

| # | 风险 | 影响 | 建议 |
|---|------|------|------|
| R1 | 路由嵌套架构 | 需要在现有路由树中嵌入 `/admin/*` 子路由，同时隔离布局 | Alpha 优先验证 `<Outlet>` 嵌套布局模式 |
| R2 | 侧边栏折叠状态管理 | 全局状态同步 (Sidebar 折叠 ↔ 内容区宽度) | 使用 React Context 或简单 useState 提升 |
| R3 | Mock 数据层设计 | 需支持 CRUD 操作并在页面间保持状态一致 | 使用 React Context + useReducer 模拟数据存储 |
