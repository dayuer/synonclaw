## 迭代复盘 — Admin 管理后台 v1.0

### 做得好的
- 架构隔离干净：admin 代码完全独立于官网，仅修改 `App.tsx` 一个现有文件
- 设计系统复用优良：admin.css 全部基于 tokens.css 变量，视觉一致性极高
- Mock 数据层设计简洁：模块闭包 > Context，避免了过度工程化
- OpenClaw RPC 研究与预留：基于现有 `openclaw.py` 路由进行设计，接口真实可用
- 浏览器验证全通过：6 个页面 + 交互功能均正常

### 需要改进的
- 缺少 Vitest 自动化单元测试（仅依赖 tsc + 浏览器验证）
- 产品管理没有删除功能
- 客户管理没有新增/编辑功能
- 移动端响应式虽有 CSS 但未做浏览器实际测试

### 行动项 (沉淀到 lessons.md)
- 大型前端模块新增时，优先验证路由嵌套方案后再写页面
- Mock 数据层用模块闭包比 Context 更简洁，但后续对接 API 时需统一替换为 React Query 等方案
- 管理后台类项目，表格/表单/徽章三个通用组件占据 80% 复用率，应优先抽象
