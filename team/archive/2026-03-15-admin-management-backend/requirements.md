# SynonClaw Admin 管理后台

> 🎯 PM 产出 | 复杂度：**L (复杂)** — 新增独立子系统，涉及全新路由体系、布局架构、认证机制与多模块 CRUD

## 背景

SynonClaw 定位为企业级软硬一体数字员工终端。当前项目仅有面向公众的官网（5 个展示页面）。需要新增一个 **Admin 管理后台**，承载商业运营所需的核心管理功能：产品管理、客户管理、订单管理、开发者网络管理、系统监控。

## 用户故事

### US-1: 管理后台基础架构

**作为** 系统管理员
**我想要** 通过独立的 `/admin` 路由访问管理后台
**以便于** 与公众网站逻辑隔离，独立管理业务数据

#### AC (验收标准)

- AC-1.1: `/admin` 路由加载独立布局（侧边栏 + 顶栏 + 内容区），不复用官网 Navbar/Footer
- AC-1.2: 侧边栏导航包含所有管理模块入口，当前模块高亮
- AC-1.3: 顶栏显示系统名称、管理员名称、退出按钮
- AC-1.4: 整体设计延续暗色主题，复用 `tokens.css` 设计系统

### US-2: Dashboard 仪表盘

**作为** 系统管理员
**我想要** 看到全局运营数据概览
**以便于** 快速掌握业务核心指标

#### AC (验收标准)

- AC-2.1: 四个统计卡片 — 设备总数、企业客户数、活跃订单、开发者数量
- AC-2.2: 卡片支持数字动态效果（计数器动画）
- AC-2.3: 显示最近订单列表（表格，最新 5 条）
- AC-2.4: 显示系统状态概览（在线设备率、API 调用量等指标）

### US-3: 产品管理

**作为** 系统管理员
**我想要** 管理 SynonClaw 硬件产品线
**以便于** 维护产品信息和定价

#### AC (验收标准)

- AC-3.1: 产品列表页 — 表格展示产品名、类型（桌面级/机柜）、规格摘要、价格、状态
- AC-3.2: 产品详情/编辑 — 表单编辑产品名、描述、规格参数、定价、上架状态
- AC-3.3: 新增产品操作（复用编辑表单）
- AC-3.4: 产品状态切换（上架/下架）

### US-4: 客户管理

**作为** 系统管理员
**我想要** 管理企业客户信息
**以便于** 跟进客户生命周期

#### AC (验收标准)

- AC-4.1: 客户列表 — 企业名、联系人、类型（ToB/ToC）、状态（潜在/已签约/流失）、设备数
- AC-4.2: 客户详情 — 展示详细信息 + 关联设备列表 + 订单历史
- AC-4.3: 搜索过滤 — 按名称搜索、按类型/状态筛选

### US-5: 订单管理

**作为** 系统管理员
**我想要** 追踪硬件订单全流程
**以便于** 管理从下单到交付的完整链路

#### AC (验收标准)

- AC-5.1: 订单列表 — 订单号、客户名、产品、金额、状态、时间
- AC-5.2: 订单状态流转 — 待确认 → 生产中 → 已发货 → 已完成
- AC-5.3: 订单详情 — 完整订单信息 + 状态变更历史

### US-6: 开发者网络

**作为** 系统管理员
**我想要** 管理认证开发者
**以便于** 维护人才储备池

#### AC (验收标准)

- AC-6.1: 开发者列表 — 姓名、学校、技能标签、认证等级、状态
- AC-6.2: 开发者详情 — 个人信息 + 认证历史 + 任务记录
- AC-6.3: 认证等级变更操作

### US-7: 系统设置

**作为** 系统管理员
**我想要** 管理系统基础配置
**以便于** 自定义系统行为

#### AC (验收标准)

- AC-7.1: 系统信息展示页（版本号、环境、构建时间等）
- AC-7.2: 主题外观设置（暂以展示为主，预留扩展接口）

## 技术约束

- 前端纯客户端渲染（无后端 API，使用 Mock 数据）
- 复用现有技术栈：React 19 + TypeScript + Vite 6 + Vanilla CSS
- 复用 `tokens.css` 设计系统，扩展 admin 专用 token
- 路由集成到现有 `react-router-dom v7` 路由树
- 组件独立于官网页面，admin 目录隔离
