# 🎯 PM 微需求切割 — SynonClaw 官网

## 核心业务目标

构建 SynonClaw 软硬一体终端的官方展示与销售门户网站，包含以下转化目标：
1. **ToC**：引导极客/开发者完成桌面级硬件的直接预定
2. **ToB**：引导企业决策者填写表单预约演示
3. **开发者**：引导高校学生注册培训认证

## 微需求清单

### MR-01：项目脚手架与设计系统
- 初始化 Vite + React + TypeScript 项目
- 建立全局 CSS 设计系统（色彩/字体/间距/动画 Token）
- 响应式断点体系（桌面 / 平板 / 移动端）

### MR-02：全局导航与布局骨架
- 顶部固定导航栏（首页 / 个人版 / 企业版 / 软件生态 / 开发者网络）
- 移动端汉堡菜单
- 全局 Footer（公司信息/社交链接/备案号）
- React Router 页面路由

### MR-03：首页 — Hero Section
- 深色背景 + 双硬件渲染图并排
- 主标题/副标题文案
- 双 CTA 分流按钮（了解桌面版 / 获取企业方案）
- 入场动画（渐入 + 微位移）

### MR-04：首页 — 核心特性区
- 三列特性卡片（数据安全 / 开箱即拉群 / 人机协同）
- 图标 + 标题 + 描述
- 滚动渐入动画

### MR-05：个人版 (Desk) 页面
- Hero section + 产品渲染图
- 硬件配置清单（表格/网格化展示）
- 应用场景区块（代码辅助 / 算力自由）
- "立即预定" CTA 按钮

### MR-06：企业版 (Enterprise) 页面
- Hero section + 机柜渲染图
- 三大企业级特性区块（热插拔 / 应用自动生成 / 网络隔离）
- "预约专属演示"触发线索收集表单模态框

### MR-07：ToB 线索收集表单
- 模态框表单（企业名称 / 需求节点数 / 联系人 / 电话 / 邮箱）
- 前端表单验证
- 提交成功反馈状态

### MR-08：开发者网络页面
- Hero section
- 三步成长路径（学习 → 认证 → 接单）
- "注册开发者账号" CTA

### MR-09：软件生态展示区（含流程图）
- OpenClaw → 云兔 工作流程图
- 可视化数据流向动画

### MR-10：响应式适配与交互打磨
- 全页面移动端适配
- 微交互动画（hover / scroll reveal / 按钮反馈）
- SEO 元数据配置
