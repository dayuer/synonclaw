# GNB 操作面板需求

> PM 产出 · 复杂度 M · 2026-03-16

## 背景

现有 `NetworkPage.tsx` 是纯只读展示（节点列表 + 隧道列表）。用户需要将其升级为完整的 GNB **操作面板**，支持：节点注册、状态监控、Passcode Web 编辑、私域子网规划。

---

## 用户故事

### US-1 节点注册

**作为**管理员，**我希望**在 Web 界面上注册新的 GNB 节点（输入 UUID、虚拟 IP、节点类型、加密方式、Passcode），**以便**新设备能加入 GNB 网络。

**AC:**
- AC-1.1: 节点 Tab 新增「注册节点」按钮，弹出模态框
- AC-1.2: 表单包含：UUID(8位hex)、虚拟IP(10.1.0.x)、节点类型、加密类型、初始Passcode
- AC-1.3: 注册后新设备出现在节点列表中，状态为 online
- AC-1.4: UUID 和虚拟 IP 不可与现有节点重复（唯一性校验）

### US-2 状态监控增强

**作为**管理员，**我希望**在节点详情中看到实时状态指标（上线时间、关联隧道数、NAT 类型），**以便**快速定位故障节点。

**AC:**
- AC-2.1: 节点详情面板展示全部 GNB 属性（已有）
- AC-2.2: 节点详情展示关联隧道数量和健康状态汇总
- AC-2.3: 安全警告仍然在详情面板显示（已有）

### US-3 Passcode Web 控制

**作为**管理员，**我希望**通过 Web 界面修改节点的 Passcode，**以便**在不登录物理设备的情况下轮换安全凭证。

**AC:**
- AC-3.1: 节点详情中 Passcode 字段旁有「编辑」按钮
- AC-3.2: 点击后弹出编辑框，输入新 Passcode（8 字符 hex，格式 0xXXXXXXXX）
- AC-3.3: 提交后 Passcode 更新生效，列表/详情同步刷新
- AC-3.4: Passcode 展示时始终脱敏（已有 maskPasscode）

### US-4 私域子网规划

**作为**管理员，**我希望**创建独立的私域子网（定义子网名称、CIDR 段、Passcode），**以便**将不同客户/部门的节点隔离到不同的虚拟网络。

**AC:**
- AC-4.1: 网络页新增第三个 Tab「私域」，展示已规划子网列表
- AC-4.2: 可新建私域（名称、CIDR 如 10.1.0.0/24、子网 Passcode）
- AC-4.3: 每个私域卡片展示名称、CIDR、成员节点数、Passcode(脱敏)
- AC-4.4: 可删除空的私域（无成员节点时可删）
- AC-4.5: 可将现有节点分配到私域（通过虚拟 IP 所属 CIDR 自动匹配）
